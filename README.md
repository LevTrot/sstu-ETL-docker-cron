# БЫСТРЫЙ README

PostgreSQL + MongoDB, код на Python в несколько скриптов (лежат по папкам). 

в Постгресе у нас бд на 10.000 покупателей (customers), и рандомно они совершили 500.000 покупок(orders), так же 10.000 товаров(products). Запускать генерацию псевдо-данных надо вручную, она автоматически нигде не активируется.

Cron - раз в минуту выполняет репликацию. Создается из entrypoint.sh, только такой способ придумал для второго задания, чтобы передать .env.

Доп.задания:

1. добавлена таблица products, а так же order_products, для реализации многие-ко-многим(в монго сохраняем как вложенные массивы). 

2. все подключения к бд и интервалы логирования находятся в .env файле. Подключал через os в питоне и через .sh файл для cron.

3. отслеживаение удаления реализовано. Есть пустая строка deleted_at, которая заполняется при удалении элемента и это заполнение переносится в монгодб (фактического скрипта .py или sql для удаления в работе не писал, проверял лишь вручную в интерфейсе бд-шки, soft delete ведь подразумевает, что мы просто помечаем как удаленный, а не буквально сносим записи, так-что я просто присваивал NOW() к deleted_at).

4. идемпонентность реализована. Cron не дублирует элементы, так как используется collection.replace_one({"_id": doc["_id"]}, doc, upsert=True). То есть либо изменяем существующий, либо создаем новый в случае отсутствия элемента по id. Это у нас в replicate.py код.
